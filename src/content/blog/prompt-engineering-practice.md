## プロンプトエンジニアリングとは

プロンプトエンジニアリングとは、生成AIに最適な指示を与えるための「言葉の設計技術」です。

同じAIでも、プロンプトの書き方で出力品質が大きく変わります。

**例: 議事録作成**

❌ 悪いプロンプト:
```
議事録を作って
```

✅ 良いプロンプト:
```
以下の会議音声から議事録を作成してください。

【出力形式】
- 日時・参加者
- 議題ごとの要点（箇条書き）
- 決定事項
- 次回までのアクション（担当者・期限）

【注意】
- 発言者名を明記
- 専門用語はそのまま使用
- 500文字以内
```

## 初級者と上級者の違い

**初級者:** 「やってもらう指示」を書く

**上級者:** 「生成物を設計する意識」で作成する

この意識の転換が、品質向上の鍵です。

## 効果的なプロンプトの4要素

### 1. Instruction（命令）

AIに何をしてほしいかを明確に指示します。

```
以下の文章を要約してください
このコードのバグを特定してください
顧客向けのメール文を作成してください
```

### 2. Context（文脈）

背景情報を与えることで、精度が向上します。

```
あなたは10年の経験を持つシステムエンジニアです。
このシステムは中小企業の在庫管理に使われています。
読者はITに詳しくない経営者です。
```

### 3. Input Data（入力データ）

処理対象のデータを提供します。

```
【処理対象】
売上データ: ...
顧客リスト: ...
```

### 4. Output Indicator（出力形式）

どのような形式で出力してほしいか指定します。

```
【出力形式】
- JSON形式で
- 箇条書きで3点
- 表形式で比較
- 500文字以内
```

## 実務で使えるテクニック

### Zero-shot プロンプティング

回答例なしで指示する方法。シンプルなタスクに有効。

```
以下の文章を英語に翻訳してください。
「本日はお忙しい中、ありがとうございます」
```

### Few-shot プロンプティング

良い回答例を示すことで、一貫した品質を維持。

```
以下の形式で顧客対応メールを作成してください。

【例】
件名: お問い合わせありがとうございます
本文: 〇〇様
この度はお問い合わせいただき...

【作成依頼】
顧客名: 山田様
内容: 納期の確認
```

### Chain of Thought（思考の連鎖）

複雑な問題を段階的に解かせる。

```
この問題を以下の手順で解いてください。
1. まず問題を分解する
2. 各部分を分析する
3. 結論を導く
```

### ロール設定

AIに役割を与える。

```
あなたはセキュリティの専門家です。
以下のコードの脆弱性を指摘してください。
```

## 開発現場での活用例

### コードレビュー

```
以下のコードをレビューしてください。

【観点】
- セキュリティ脆弱性
- パフォーマンス
- 可読性
- エラーハンドリング

【出力形式】
問題点ごとに「重要度（高/中/低）」「該当箇所」「改善案」を記載
```

### テストケース生成

```
以下の関数のテストケースを作成してください。

【条件】
- 正常系・異常系両方
- 境界値テストを含む
- Jest形式で出力

【対象関数】
(関数コード)
```

### ドキュメント作成

```
以下のAPIエンドポイントの仕様書を作成してください。

【含める情報】
- エンドポイントURL
- HTTPメソッド
- リクエストパラメータ
- レスポンス形式
- エラーコード

【対象】
(APIコード)
```

## 日本語プロンプトの注意点

生成AIの多くは英語環境で開発されているため、日本語特有の課題があります。

- 主語の省略が多い → 主語を明示する
- 文脈依存性が高い → 背景情報を丁寧に記述
- 曖昧な表現が多い → 具体的に指定する

## まとめ

プロンプトエンジニアリングは、AI活用の精度を左右する重要な技術です。

**ポイント:**
1. 「指示」から「設計」への意識転換
2. 4要素（命令・文脈・入力・出力形式）を意識
3. 目的に応じたテクニックの使い分け
